# Security Policy

## Supported Versions

Currently, only the latest version of the project is actively supported with security updates. 

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

Please **do not** report security vulnerabilities through public GitHub issues. 

If you discover a security vulnerability, please report it privately via email to:
**rovaldavc@gmail.com**

### Response Timeline

* I will do my best to acknowledge receipt of your report within **48 hours**.
* Following triage, I will keep you updated on the progress of the fix and the expected timeline for a patch release.

### Scope

**In Scope:**
* Authentication or authorization bypasses
* Unauthorized access to private chat messages or user data
* Injection attacks (SQLi, Command Injection, etc.)

**Out of Scope:**
* Volumetric Denial of Service (DoS/DDoS) attacks against live infrastructure
* Spam, phishing, or social engineering attacks against users
* Issues requiring physical access to a user's device

## Recognition

While this project does not currently offer a paid bug bounty, I deeply appreciate the efforts of security researchers. Valid reports that lead to a fix will be recognized with public credit and appreciation in the project's release notes and security advisories.
